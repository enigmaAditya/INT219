1. The CSS Box Model

Every HTML element is a rectangular BOX with 4 layers: Content → Padding → Border → Margin (inside → outside)

1.1 Interactive Box Model Visualizer

Think of it like a package:
Content = the item inside the box
Padding = bubble wrap around the item (INSIDE the box)
Border = the box walls themselves
Margin = space between this box and other boxes on the shelf

Key Rule: box-sizing: border-box makes width/height INCLUDE padding and border. Without it, they're ADDED on top.

1.2 Margin Collapse

When two vertical margins touch (top margin meets bottom margin), they DON'T add up — the larger one wins.
Box 1 has margin-bottom: 30px, Box 2 has margin-top: 20px.
Space between them = 30px (NOT 50px!). The 30px "absorbs" the 20px.

⚠️ This ONLY happens vertically, between block elements, and NOT in flex/grid containers.
Box 1 — margin-bottom: 30px
Box 2 — margin-top: 20px

↕ Actual gap = 30px (larger wins), NOT 50px

1.3 Padding & Margin Shorthand

Both padding and margin use the same shorthand patterns:

padding: 20px; → all 4 sides = 20px
padding: 20px 40px; → top/bottom = 20px, left/right = 40px
padding: 10px 20px 30px; → top = 10, left/right = 20, bottom = 30
padding: 10px 20px 30px 40px; → top right bottom left (clockwise → TRouBLe)

Remember: T-R-B-L = Top Right Bottom Left = TRouBLe

2. The Display Property

Controls HOW an element behaves in the document flow — block, inline, inline-block, flex, grid, or none.

2.1 Block vs Inline vs Inline-Block

block → takes FULL width, starts on new line, CAN set width/height
inline → only as wide as content, same line, CANNOT set width/height
inline-block → sits in line like inline, but CAN set width/height (best of both!)
none → completely removed from page (no space taken)

Default block elements: div, p, h1-h6, section, header, footer, ul, form
Default inline elements: span, a, strong, em, img, input

3. Flexbox (Flexible Box Layout)

1-dimensional layout system — arranges items in a ROW or COLUMN. Perfect for navbars, centering, card rows, and component layouts.

3.1 flex-direction — Which way items flow

row (default) → items go LEFT to RIGHT →
row-reverse → items go RIGHT to LEFT ←
column → items go TOP to BOTTOM ↓
column-reverse → items go BOTTOM to TOP ↑

Important: When direction changes, the MAIN AXIS and CROSS AXIS swap!
Row: main = horizontal, cross = vertical
Column: main = vertical, cross = horizontal

3.5 gap — Space between items

gap: 20px → 20px space between ALL items
row-gap: 20px → vertical gap only
column-gap: 30px → horizontal gap only
gap: 20px 30px → row-gap column-gap

Why gap is better than margin: Gap only creates space BETWEEN items, not around the edges. With margin, you'd get extra space on the first/last items.

3.6 flex (Child Property) — How items grow and shrink

flex: grow shrink basis

flex-grow → how much this item should GROW to fill extra space (0 = don't grow)
flex-shrink → how much to SHRINK when space is tight (0 = don't shrink)
flex-basis → initial size before growing/shrinking

Common shortcuts:
flex: 1 → grow equally (all items share space evenly)
flex: 2 → take TWICE the space of flex: 1 items
flex: 0 0 300px → don't grow, don't shrink, stay 300px (fixed sidebar)
flex: 1 1 300px → grow/shrink, start at 300px (responsive card)

4. CSS Grid (2-Dimensional Layout)

Grid handles rows AND columns simultaneously. Use Grid for page layouts, card grids, and any 2D arrangement.

4.1 grid-template-columns — Define column widths

grid-template-columns: 200px 200px 200px → 3 fixed columns
grid-template-columns: 1fr 1fr 1fr → 3 equal flexible columns
grid-template-columns: 1fr 2fr 1fr → middle is twice as wide
grid-template-columns: 250px 1fr → fixed sidebar + flexible main
grid-template-columns: repeat(3, 1fr) → shorthand for 1fr 1fr 1fr
grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)) → ✨ RESPONSIVE MAGIC!

fr unit = "fraction" of available space. 1fr 2fr 1fr divides space into 4 parts: 1 part, 2 parts, 1 part.

4.7 Flexbox vs Grid — When to Use Which

Use FLEXBOX when:
✅ Navigation bars (items in a row)
✅ Centering content
✅ Items with VARYING sizes
✅ Single row OR column layouts
✅ Space distribution (space-between)

Use GRID when:
✅ Overall page structure (header, sidebar, main, footer)
✅ Card grids with EQUAL sizing
✅ Image galleries
✅ 2D layouts (need both rows AND columns)
✅ Complex layouts with overlapping

You can NEST them: Grid for page layout → Flexbox inside each grid area for component layout.

5. CSS Positioning

Controls WHERE an element is placed — static, relative, absolute, fixed, or sticky.

5.1 Position Types — Visual Comparison

static (default) → normal flow, top/left/right/bottom have NO effect
relative → offsets from normal position, original space is KEPT
absolute → removed from flow, positioned relative to nearest positioned ancestor
fixed → removed from flow, positioned relative to VIEWPORT, stays on scroll
sticky → normal until scroll point, then fixed until parent scrolls away
