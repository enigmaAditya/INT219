HTML5

HTML (HyperText Markup Language) is the skeleton of every web page. We use semantic elements like <header>, <nav>, <main>, <section>, <article>, and <footer> to structure content meaningfully.

CSS3

CSS (Cascading Style Sheets) is the skin of the website. We use Flexbox for one-dimensional layouts, Grid for two-dimensional layouts, and media queries for responsive design across all screen sizes.

JavaScript

JavaScript adds interactivity — the brain of the website. We handle events like clicks, form submissions, and keyboard input. We manipulate the DOM to create dynamic, interactive experiences.

FAQ

Frequently asked questions

Semantic HTML uses meaningful tags like <header>, <nav>, <main>, <article>, <aside>, and <footer> instead of generic <div> tags. This helps search engines, screen readers, and developers understand the page structure.

Every HTML element is a box with four layers: Content (your text/image), Padding (space inside border), Border (the visible edge), and Margin (space outside border). Always use box-sizing: border-box so width includes padding and border.

Use Flexbox for one-dimensional layouts (items in a row OR column) — navbars, centering, button groups. Use Grid for two-dimensional layouts (rows AND columns) — page layouts, card grids, dashboards. You can nest them: Grid for page layout, Flexbox inside each area.

Instead of adding event listeners to every child element, add ONE listener to the parent and use e.target to determine which child was clicked. Benefits: fewer listeners (better performance), works for dynamically added elements.
